# 软件的主要功能

本软件全称为「**湖北数创数字产业研究期刊发布与内容管理平台**」（以下简称「本平台」），是面向湖北数字产业研究机构建设的集**数字产业内容生产、智能辅助写作、全文检索发布、AI 智能问答、运营管理后台**于一体的综合性内容管理与发布平台。系统采用前后端分离架构，前端基于 React 18 + TypeScript + Vite 构建用户访问站点与运营后台界面，后端基于 FastAPI + SQLAlchemy + SQLite 实现 RESTful API 服务，覆盖研究机构从内容采编、多媒体素材管理、AI 智能排版到对外发布的完整业务链路。

**一、研究文章与资讯发布**
平台支持研究文章的全生命周期管理，运营人员可在后台创建、编辑、删除文章，并对文章进行草稿、发布、撤回等状态流转。公开接口 `GET /api/articles` 与 `GET /api/articles/{slug}` 仅返回 `status='published'` 的文章，实现"草稿与公开"严格隔离。每篇文章支持 Markdown 格式正文、封面图、作者、阅读时长、分类、标签、精选等结构化字段。文章详情页通过 slug 访问，避免 ID 外泄与 SEO 友好。除长文外，平台还提供"前沿资讯"短消息模块 `GET /api/insights`，配合分类筛选与展开折叠交互。

**二、期刊与期号管理**
平台为机构提供完整的期刊发布能力，支持以"期刊（Journal）+ 期号（Issue）+ 文章"三级结构组织研究产出。期刊与期号同样支持草稿/发布状态管理，期刊详情页按时间倒序列出所有期号；期号详情页按文章顺序号展示本期收录的全部文章。已发布文章的 slug 在发布后不可修改，避免外链失效。

**三、Markdown 编辑与 AI 智能排版**
文章编辑器集成 Markdown 实时预览，所见即所得；支持表格、图片、超链接、代码块等富文本能力。后端 `markdown_typesetter` 服务对接大语言模型，将原始 Markdown 自动规范化为符合平台发布规范的标准化排版结果，并支持在管理端"排版预览"对话框中比对原文与排版结果。配套的 `markdown_normalize` 服务对来自 Word 的 DOCX 残留样式（多余空行、表格噪音、断行空白等）进行确定性清洗，避免再次依赖 LLM 引入不确定性。

**四、Word 文档智能导入**
`docx_import` 服务支持将 `.docx` 文件批量导入为文章草稿，使用 pandoc 完成文档结构转换，再由 `markdown_normalize` 完成内容净化，最终生成可继续编辑的标准 Markdown，大幅降低研究人员的录入成本。

**五、媒体库与封面图智能生成**
所有上传的图片集中存放于 `Media Library`，支持浏览、检索、复制 URL、删除、引用关系追溯等管理能力；删除时自动检测是否有文章仍引用该图，防止误删引发 404。平台还集成 AI 封面图生成能力，可根据文章标题与摘要自动生成符合品牌调性的封面候选图，并通过覆盖评估规则写入到 `cover_image` 字段，提升内容视觉一致性。

**六、公开站点与全文检索**
前台站点基于 React Router v6 实现单页应用，主要页面包括首页（Hero + 研究领域 + 精选文章 + 资讯流 + 团队）、研究文章列表与详情、前沿资讯、研究案例、关于我们、期刊与期号、搜索、404 五大模块。`/api/search?q=` 提供跨文章与资讯的全文检索能力，搜索结果页支持关键词高亮与分页。响应式设计适配桌面、平板与手机三种视口。

**七、AI 智能问答（Page Agent）**
平台在公开页面右下角浮出"问 AI"入口，用户点击后弹出对话面板，可针对当前页面内容向 AI 提问；后端 `public_agent_router` 携带页面正文快照、Referer 同源校验、限流策略等安全控制，并通过 SSE 流式响应逐字回传答案，过程中支持打断、复制、Markdown 渲染与代码高亮。同时后台内置管理员专用的 `agent_router`，供编辑团队完成 AI 排版、生成摘要等任务。

**八、运营管理后台（CMS）**
后台采用单管理员 + JWT 认证模型，登录、签发、刷新、注销全流程覆盖；密码使用 bcrypt 哈希存储，敏感配置（SECRET_KEY、上传目录、限流阈值等）通过 `.env` 注入。后台管理覆盖文章 CRUD、期刊 CRUD、精选文章编排、媒体库管理、系统设置、登录审计等。配套中间件 `rate_limit` 对登录、敏感接口进行 IP 级与账号级双层限流，抵御暴力破解。

**九、订阅与读者互动**
平台首页与文章详情页内置邮件订阅表单，调用 `POST /api/newsletter` 完成订阅；订阅数据落地后可用于后续电子期刊推送。

**十、可观测性与运行支撑**
后端使用 pydantic-settings 统一管理配置，结构化 JSON 日志覆盖访问、错误与慢请求；前端通过 TanStack Query 管理服务端缓存与乐观更新，关键操作（保存、排版、上传）通过自定义 Toast 提供明确反馈；全局 `ErrorBoundary` 兜底渲染错误，避免白屏。

综上，本平台通过模块化、智能化与可运营的设计，覆盖了数字产业研究机构从内容生产到对外发布的全流程需求。
